# Security Policy

## Supported versions

Pre-1.0 releases are best-effort for security patches.

## Reporting a vulnerability

- Report privately to project maintainers.
- Include impact, reproduction steps, and affected versions.
- Expect an acknowledgement within 72 hours.

## Disclosure process

- Confirm the issue and assess severity.
- Prepare a fix and release.
- Publish an advisory with upgrade guidance.

## Scope

This policy applies to code in this repository and official release artifacts.
